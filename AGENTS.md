# Agent Handbook for Ory Talos

## 📦 Project overview
- **Language**: Go 1.26 (module `github.com/ory/talos`).
- **Purpose**: High‑performance API‑key service with admin & self‑service APIs, JWT/macaroons derivation, multi‑DB support.
- **Key packages**:
  - `cmd/` – CLI entry points (`talos`, sub‑commands).
  - `internal/` – core implementation, persistence, crypto, tracing, metrics.
  - `pkg/` – public SDKs & helper libraries.
  - `commercial/` – enterprise‑only code guarded by build tags.
  - `oryx/` – vendored Ory‑X utilities (replace directive in `go.mod`).

- **Go client SDK**: `internal/client/generated` – regenerated by `make generate`.
- **Docs**: `make docs-test` validates markdown examples, `make generate` syncs code blocks.

## 🗄️ Persistence layer (core contract)
- Interface `persistence.Persister` (see `internal/persistence/persistence.go`).
- **Network ID (NID)** handling:
  - OSS → `uuid.Nil` (single‑tenant).
  - Commercial → `contextx.RequiredNetworkIDFromContext` – fails loudly if missing.
- Drivers live under `internal/persistence/sqlc/generated` (SQLite, Postgres, MySQL, Cockroach).  
- **Gotcha**: background workers must inject NID into the context before calling any persister method.

## 🔧 Code conventions (enforced by CI & lint config)
- No global state – all dependencies are injected.
- Always pass `context.Context` from the caller.
- Structured JSON logging via `slog` (see `internal/logger`).
- Use the injected `*retryablehttp.Client` – direct `http.Get/Do` is forbidden (`golangci-lint` `forbidigo`).
- Avoid `COUNT(*)`, `SELECT *`, and `OFFSET`; use explicit column lists and cursor pagination.
- Commercial code must be under `/commercial/` and guarded with `//go:build commercial`.
- Tests must be table‑driven, parallel where safe, and cover happy & error paths (`make test` expects ≥85% coverage).

## 🧪 Testing setup
- **Docker Compose** files:
  - `docker-compose.oss.yaml` – SQLite + optional DB containers for OSS.
  - `docker-compose.test.yml` – PostgreSQL, MySQL, Cockroach test DBs.
- Run `make test-db-start` before `make test` if you need to inspect the DB.
- Integration tests live under `test/` (E2E) and use the same Docker Compose stack.
- Lint exclusions exist for generated protobuf (`*.pb.go`) and test helper patterns – respect them.

## 📦 CI / GitHub Actions
- `ci.yaml` runs lint, test (OSS), Docker Compose verification, and a demo step.
- Commercial docs are **not** built on the OSS mirror (see comments in `ci.yaml`).
- Pull‑request validation uses the same `make verify` pipeline.

## ⚠️ Gotchas & non‑obvious patterns
1. **Network ID injection** – commercial code crashes if the NID is missing; remember to call `contextx.WithNetworkID(ctx, nid)` in background workers.
2. **Build tags** – the `commercial/` folder is excluded from OSS builds; adding a file there without `//go:build commercial` will break `make build`.
3. **HTTP client** – direct `http.Get/Post` triggers a `forbidigo` lint error; always obtain the client via `internal/client.NewClient()` which returns a retryable client.
4. **SQL generation** – `sqlc` runs on both `internal/persistence/sqlc` (OSS) and `commercial/persistence/sqlc` (enterprise). Changing a schema requires running `make generate` to keep both in sync.
5. **Protobuf generation** – `buf` excludes generated files with `// Code generated by protoc` comments; never edit those files directly.
6. **Lint config** – many linters are disabled for readability (e.g., `lll`, `varnamelen`). Do not enable them globally without a strong reason.
7. **Testing** – tests deliberately ignore `COUNT(*)` style queries; the linter `gosec` excludes false‑positives for DB calls.

## 📦 Useful make shortcuts
```
make deps           # install all required tools
make generate      # regenerate protobuf, sqlc, OpenAPI, docs, SDK
make fmt           # format all sources
make lint          # run linters
make test           # run unit & integration tests (needs DB containers)
make verify        # fmt + lint + test + docs-test (full CI check)
make build          # build OSS binary
make build-commercial # build enterprise binary
```
